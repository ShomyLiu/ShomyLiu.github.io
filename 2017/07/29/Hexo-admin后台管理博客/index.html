Hexo-admin后台管理博客

文章目录
  1. 需求
  2. 解决方法

需求

Hexo这种静态博客虽然用起来很方便，但是硬伤就是没有后台管理，每次想写博客或者说想要修改原博客的话，都需要在本机上找markdown文件，修改之后发布，尤其是如果机器上没有安装hexo,git等工具的时候，就无法修改了。因此hexo可不可以以像wp,ghost这种有后台管理的话。 可能这样的需求比较大， hexo专门有一个插件hexo-admin 就是做这个工作的，提供了web 增删改查博客的功能，下面就大致记录下自己使用的hexo-admin时候遇到的一些坑。

解决方法

首先说一下条件与需求。 我自己有一个的VPS，本来是可以直接将hexo部署到VPS的，但是考虑到自己维护的VPS的稳定性等问题，最后还是本地写博客，然后把hexo产生的静态页面推送到github-pages上。

我的目的有两个:

  • 拥有在线编辑博客的后台
  • 本地机器同时也可以编辑发布博客

基于这样的需求，就需要在线VPS的内容与本地机器上的博客的资源要保持一致。就是说我在线修改了某篇博客，然后之后如果可以很简单的同步到本地，使得两端保持同步，反过来一样。

我的做法是:

  • 将hexo目录下的相关文件包括_config.yml,source/_posts的所有md文件等建立一个私人repo(使用[coding.net])
  • VPS上安装好hexo，然后可以将本地博客目录直接copy到VPS上，不用去一个一个重新安装插件了。
  • VPS上安装hexo-admin插件，这样可以在线编辑了(可以解析一个子域名+Nginx做前端转发)
  • 在博客目录写一个git同步脚本:server-upload.sh 用来同步在线博客目录下的变化，基本过程先git pull之后push

  • 建一个hexo_deploy.sh的脚本,内容是

  • 利用hexo-admin的deploy功能，可以在线发布，发布的过程基本就是hexo-deploy.sh的内容，首先需要修改_config.yml加上hexo-admin的admin选型，然后加一个字段: deployCommand: sh
  • 编辑完博客时候，发布的时候，需要在hexo-admin的deploy的输入框内写上hexo-deploy.sh即可。其中执行脚本的原理是调用了nodejs的spawn函数。

  • 本地也需要有一个upload.sh的脚本, 每次同样需要先git pull 在git push 到私人repo,还要多一个过程是将最新的内容pull给VPS。

大致过程就是这样，其中坑点就是部署的时候的spawn一直报错，后来采用上述deployCommand的方式才ok。具体的细节就不再赘述。 需要脚本的可以邮箱或者评论联系。
